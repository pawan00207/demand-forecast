Problem Statement

Retail businesses face significant challenges in predicting customer demand accurately. Over-stocking leads to wastage; under-stocking causes lost sales.

Objective: Build a regression model that predicts weekly/daily sales for a given product-store combination based on features like price, discount, advertising spend, seasonality, and holidays.

Business Impact: Every 1% improvement in forecast accuracy can reduce inventory costs by 2–3% and increase revenue by 1–2%.

Mathematical Foundation
Model Form:
Ŷ = β₀ + β₁X + ε

Least Squares Estimation:
β₁ = Σ[(xᵢ−x̄)(yᵢ−ȳ)] / Σ[(xᵢ−x̄)²]
β₀ = ȳ − β₁x̄

Goodness of Fit (R²):
R² = 1 − (SS_res / SS_tot)
Range: [0, 1] — higher is better
Mean Error (Custom Metric):
ME = Σ(actual − predicted) / n

Limitation of SLR:
• Uses only 1 feature — misses many signals
• Cannot capture interaction effects
• Assumes linear relationship

Why Use MLR Next?
MLR uses all available features simultaneously, dramatically improving prediction accuracy (R² jumps from ~0.41 to ~0.70).

Logistic Regression — Brief Explanation
What is Logistic Regression?
While Linear Regression predicts continuous values (sales units), Logistic Regression predicts probabilities for binary outcomes.

Demand Forecasting Use Case:
P(stockout) = σ(β₀ + β₁×demand + β₂×inventory)

Predict: Will demand EXCEED supply? (Yes/No)
Sigmoid Function:
σ(z) = 1 / (1 + e^(-z))

Maps any real value → probability [0, 1]

Applications in Retail:
• Predicting if a product will sell out (0/1)
• Classifying high vs low demand days
• Fraud detection in sales data
